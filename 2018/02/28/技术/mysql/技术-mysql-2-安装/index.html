技术-mysql-2.安装

发表于 2018-02-28 | 分类于 技术 ， mysql |



安装（linux平台-RPM包）

当前主流的Linux发行版本中都已经集成mysql，可以直接安装使用，但可能版本较旧。因此，可以前往官网下载最新版本的安装介质，有rpm包和tar包，官方不建议我们使用源码包安装，直接使用rpm包安装即可。

rpm包来源方式分为两类：

  • 直接从操作系统发行版本中自带的rpm包进行安装
  • 从官方网站下载rpm包进行安装

系统自带rpm包安装

1.借助软件包管理工具如yum或apt-get可以更方便的进行二进制包的安装（帮助自动解决包的依赖关系）

3.按提示内容设置密码（没有报错则说明安装成功）

4.设置好密码后，测试登录

5.将我们的mysql服务设置成开机启动

注意：从mysql5.1开始mysql的官方下载版本的服务名称改为mysql，将不再是mysqld。而rhel自带的发行版本的mysql名称依然沿用mysqld，这点区别需要特别注意。

官网下载rpm包进行安装

1.前往官方网站下载rpm安装包，或直接在服务器上通过wget下载官方rpm安装包
MySQL官方网站：http://www.mysql.com/downloads/

2.下载后使用rpm -ivh 或yum方式安装即可

安装（linux平台-yum）

前往官方网站下载yum源的rpm包，或直接在服务器上使用wget的方式下载该包到服务器上，执行安装该rpm包，安装好该yum源的rpm包后，会自动在/etc/yum.repos.d目录下创建yum仓库的配置文件。再使用yum install mysql的方式进行安装，它就会直接从mysql的官方仓库下载并安装mysql

安装（linux平台-源码包）

使用源码通常用于开发、调试、测试环境这种需要定制的小范围内使用，通常生产环境这种大规模部署环境不推荐使用源码安装，因为这样做不仅不利于规模化批量部署，版本无法统一，碎片化严重，而且安装时间长难以控制。
使用源码安装主要是用于定制编译参数，比如：安装位置，目录位置，功能开关开启或关闭等等。

用源码安装有两个好处：

  • 自定义所需要的模块
  • 自定义安装路径

注意：以源代码包安装mysql时，如果没有写的参数，有默认值的参数会选择采取默认值，没有默认值的参数这部分就被忽略了！

新的编译三板斧
支持跨平台，从mysql5.6开始以这种方式来编译安装

编译脚本（下一步会用到）

mysql5.7版本因为需要使用boost类库，所以在编译安装时需要带上这段参数：-DDOWNLOAD_BOOST=1 -DWITH_BOOST=/usr/local/src/mysql-5.7.13/boost ，如果不带上的话编译过程中也会报错提示你带上这段参数的，所指向的boost目录是mysql已自带有boost的安装包

脚本说明：

参数项 说明
-DCMAKE_INSTALL_PREFIX=/usr/local/mysql mysql程序安装主目录
-DMYSQL_DATADIR=/var/lib/mysql mysql数据目录
-DSYSCONFDIR=/etc mysql配置文件目录
-DWITH_INNOBASE_STORAGE_ENGINE=1 以下是支持各种引擎，1是开启，0是不开启
-DWITH_ARCHIVE_STORAGE_ENGINE=1
-DWITH_BLACKHOLE_STORAGE_ENGINE=1
-DWITH_FEDERATED_STORAGE_ENGINE=1
-DWITH_PARTITION_STORAGE_ENGINE=1
特别注意分区的引擎
-DMYSQL_UNIX_ADDR=/tmp/mysql.sock
-DENABLED_LOCAL_INFILE=1 实验环境可打开，生产环境建议关闭
-DWITH_EXTRA_CHARSETS=all 扩展的字符集，支持全部

源码编译安装全过程

1.配置好网络环境和selinux和iptables
2.安装开发工具包（强烈要求，可以避免后续编译过程中报的很多依赖关系包缺失错误）

3.安装cmake

安装相关包

4.将mysql源代码安装包解压

5.将编译脚本放置在解压的安装包目录下
6.创建安装mysql的非可登录用户

7.运行cmake.sh
8.根据提示错误信息进行一些包的补充安装（mysql5.7需要有boost类库，所以参数中需要增加此参数项）
9.gmake或make install直接一步编译与安装
10.安装完成后，进入到安装主目录，即/usr/local/mysql，可查看到相应文件
11.进入/usr/local/mysql/support-files目录，将mysql.server复制到/etc/init.d/目录下作为服务，方便以后启停mysql服务
12.将/usr/local/mysql/support-files下的my.cnf拷贝到/etc/my.cnf作为mysql的主配置文件（通常发行版本中已经自带了该配置文件，可以直接覆盖替换），并进行相关的配置选项进行修改（实际上你不指定my.cnf配置文件，mysql的服务也可以启动起来，之所以指定my.cnf的原因是我们可以在配置文件中可以调配mysql的运行方式，所以可以根据你的生产环境来自定义调配）

13.将相关目录的所属用户与组设置为mysql以提供权限

14.（重要）使用源码安装的mysql，编译安装完成后需要进行初始化，才可以启动服务，切记。
到mysql的安装主目录，即/usr/local/mysql目录下执行：

15.启动mysql服务

16.将/usr/local/mysql/bin目录添加到环境变量并使之生效，以使该bin目录下的工具可用

17.安装主机使用客户端连接测试

18.可以执行/usr/local/mysql/bin目录下的mysql_secure_installation（安全初始化）或mysqladmin（为root设置密码）

安装（linux平台-SPRM包）

1.整理好网络与iptables和selinux
2.安装yum groupinstall 'Development tools'
3.前往官网在源代码部分找到srpm包并下载
4.创建用户（默认是以普通用户来构建，root用户也可以，但是需要做一些微调，为了避免不必要的麻烦，因此我们需要先创建一个普通用户）

5.将srpm包放到/home/mysql这个普通用户的家目录里
6.因为需要mysql这个普通用户来管理和使用这个包，所以修改包的权限

7.确认一下当前环境是否安装了rpm-build

8.切换到mysql这个普通用户

9.使用rpmbuild来创建出构建目录

10.该rpmbuild目录下有相应生成的文件（如：BUILD BUILDROOT RPMS SOURCES SPECS SRPMS），最后执行构建的时候需要这几个文件来存放生成的文件和一些中间文件

11.环境构建好之后，安装srpm包

12.修改/home/mysql/rpmbuild/SPECS/下的mysql.spec文件，根据自己的需要定制即可

13.执行命令rpmbuild -bb mysql.spec就会根据spec中的配置构建出rpm包来了，如果有错误提示，根据错误提示安装相应的依赖包即可。查看选项帮助：rpmbuild --help

通常需要这些依赖库包：

  • cmake
  • cmake28 （自找安装包）
  • libaio-devel
  • ncurses-devel
  • numactl-devel
  • openssl-devel

14.这个构建的过程会耗费更长的时间，因为它需要构建出源码包、SRPM包、RPM包
15.构建完成后，我们可以在/home/mysql/rpmbuild/RPMS这里查看到我们自定义构建生成的RPM包
16.然后就可以对使用rpm -ivh 或yum install的方式来安装这些构建的mysql的rpm包了。

安装（windows平台）

在windows平台下有两种安装文件类型：msi和zip包，操作方式都类似，都是双击可执行文件进行安装的。 如果是msi直接双击运行安装即可，如果是zip，双击解压缩，运行“setup.exe”。

以下说明是基于mysql 5.5之前的界面（包括5.5），5.6开始不是这种安装界面（类似不再赘述）

1.开始安装，出现如下界面，mysql安装向导启动，按“Next”继续

2.选择安装类型，有“Typical（默认）”、“Complete（完全）”、“Custom（用户自定义）”三个选项，我们选择“Custom”，有更多的选项，也方便熟悉安装过程，如果快速安装的话直接选择typical典型安装方式即可

3.在“Developer Components（开发者部分）”上左键单击，选择“This feature, and all subfeatures, will be installed on local hard drive.”，即“此部分，及下属子部分内容，全部安装在本地硬盘上”。在上面的“MySQL Server（mysql服务器）”、“Client Programs（mysql客户端程序）”、“Documentation（文档）”也如此操作，以保证安装所有文件。点选“Change…”，手动指定安装目录。

4.填上安装目录，我的是“F:\Server\MySQL\MySQL Server 5.0”，也建议不要放在与操作系统同一分区，这样可以防止系统备份还原的时候，数据被清空。按“OK”继续。

5.返回刚才的界面，按“Next”继续

6.确认一下先前的设置，如果有误，按“Back”返回重做。按“Install”开始安装。

7.正在安装中，请稍候，直到出现下面的界面

8.这里是询问你是否要注册一个mysql.com的账号，或是使用已有的账号登陆mysql.com，一般不需要了，点选“Skip Sign-Up”，按“Next”略过此步骤。

9.现在软件安装完成了，出现上面的界面，这里有一个很好的功能，mysql配置向导，不用向以前一样，自己手动乱七八糟的配置my.ini了，将 “Configure the Mysql Server now”前面的勾打上，点“Finish”结束软件的安装并启动mysql配置向导。

10.mysql配置向导启动界面，按“Next”继续

11.选择配置方式，“Detailed Configuration（手动精确配置）”、“Standard Configuration（标准配置）”，我们选择“Detailed Configuration”，方便熟悉配置过程。

12.选择服务器类型，“Developer Machine（开发测试类，mysql占用很少资源）”、“Server Machine（服务器类型，mysql占用较多资源）”、“Dedicated MySQL Server Machine（专门的数据库服务器，mysql占用所有可用资源）”，大家根据自己的类型选择了，一般选“Server Machine”，不会太少，也不会占满。

13.选择mysql数据库的大致用途，“Multifunctional Database（通用多功能型，好）”、“Transactional Database Only（服务器类型，专注于事务处理，一般）”、“Non-Transactional Database Only（非事务处理型，较简单，主要做一些监控、记数用，对MyISAM数据类型的支持仅限于non-transactional），随自己的用途而选择了，我这里选择“Transactional Database Only”，按“Next”继续。

14.对InnoDB Tablespace进行配置，就是为InnoDB 数据库文件选择一个存储空间，如果修改了，要记住位置，重装的时候要选择一样的地方，否则可能会造成数据库损坏，当然，对数据库做个备份就没问题了，这里不详述。我这里没有修改，使用用默认位置，直接按“Next”继续

15.选择您的网站的一般mysql访问量，同时连接的数目，“Decision Support(DSS)/OLAP（20个左右）”、“Online Transaction Processing(OLTP)（500个左右）”、“Manual Setting（手动设置，自己输一个数）”，我这里选“Online Transaction Processing(OLTP)”，自己的服务器，应该够用了，按“Next”继续

16.是否启用TCP/IP连接，设定端口，如果不启用，就只能在自己的机器上访问mysql数据库了，我这里启用，把前面的勾打上，Port Number：3306，在这个页面上，您还可以选择“启用标准模式”（Enable Strict Mode），这样MySQL就不会允许细小的语法错误。如果您还是个新手，我建议您取消标准模式以减少麻烦。但熟悉MySQL以后，尽量使用标准模式，因为它可以降低有害数据进入数据库的可能性。按“Next”继续

17.西文编码，第二个是多字节的通用utf8编码，都不是我们通用的编码，这里选择第三个，然后在Character Set那里选择或填入“gbk”，当然也可以用“gb2312”，区别就是gbk的字库容量大，包括了gb2312的所有汉字，并且加上了繁体字、和其它乱七八糟的字——使用mysql的时候，在执行数据操作命令之前运行一次“SET NAMES GBK;”（运行一次就行了，GBK可以替换为其它值，视这里的设置而定），就可以正常的使用汉字（或其它文字）了，否则不能正常显示汉字。按 “Next”继续。

18.选择是否将mysql安装为windows服务，还可以指定Service Name（服务标识名称），是否将mysql的bin目录加入到Windows PATH（加入后，就可以直接使用bin下的文件，而不用指出目录名，比如连接，“mysql.exe -uusername -ppassword;”就可以了，不用指出mysql.exe的完整地址，很方便），我这里全部打上了勾，Service Name不变。按“Next”继续。选择是否将mysql安装为windows服务，还可以指定Service Name（服务标识名称），是否将mysql的bin目录加入到Windows PATH（加入后，就可以直接使用bin下的文件，而不用指出目录名，比如连接，“mysql.exe -uusername -ppassword;”就可以了，不用指出mysql.exe的完整地址，很方便），我这里全部打上了勾，Service Name不变。按“Next”继续。

19.这一步询问是否要修改默认root用户（超级管理）的密码（默认为空），“New root password”如果要修改，就在此填入新密码（如果是重装，并且之前已经设置了密码，在这里更改密码可能会出错，请留空，并将“Modify Security Settings”前面的勾去掉，安装配置完成后另行修改密码），“Confirm（再输一遍）”内再填一次，防止输错。 “Enable root access from remote machines（是否允许root用户在其它的机器上登陆，如果要安全，就不要勾上，如果要方便，就勾上它）”。最后“Create An Anonymous Account（新建一个匿名用户，匿名用户可以连接数据库，不能操作数据，包括查询）”，一般就不用勾了，设置完毕，按“Next”继续。

20.确认设置无误，如果有误，按“Back”返回检查。按“Execute”使设置生效。

21.设置完毕，按“Finish”结束mysql的安装与配置——这里有一个比较常见的错误，就是不能“Start service”，一般出现在以前有安装mysql的服务器上，解决的办法，先保证以前安装的mysql服务器彻底卸载掉了；不行的话，检查是否按上面一步所说，之前的密码是否有修改，照上面的操作；如果依然不行，将mysql安装目录下的data文件夹备份，然后删除，在安装完成后，将安装生成的 data文件夹删除，备份的data文件夹移回来，再重启mysql服务就可以了，这种情况下，可能需要将数据库检查一下，然后修复一次，防止数据出错。

22.提示：如果你在安装完成后没有勾选开始配置，可以自行到安装目录下，找到bin目录，双击“MySQLInstanceConfig.exe”可以开始配置。
