存储引擎层

  • 客户端：mysql客户端并不具体指某个客户端软件，事实上mysql客户端是一种复合概念。包含：不同程序语言编写的前端应用程序，所调用的API接口
  • 事务：事务是由一组SQL语句组成的逻辑处理单元（视频说明暂不全）
    • 原子性：事务是一个原子操作单元，其对数据的修改，要么全都执行，要么全都不执行（比如：汇钱）
    • 一致性：在事务开始和完成时，数据都必须保持一致状态
    • 隔离性：数据库系统提供一定的隔离机制，保证事务在不受外部并发操作影响的“独立”环境执行。这意味着事务处理过程中的中间状态对外部是不可见的，反之亦然
    • 持久性：事务完成之后，它对于数据的修改是永久性的，即使出现系统故障也能够保持

并发控制

（多事务并发处理是需要并发控制的）
1.对并发操作进行正确调度
2.保证事物的隔离性
3.保证数据的一致性

多事务的执行方式：

  • 串行执行：同一时间只有一个事物运行，效率低，无法充分利用资源（银行只开一个窗口）
  • 交叉并发执行：多事务交叉运行，单处理器的并发方式（类似于时分多差的方式，银行只开一个窗口，利用处理的空档处理其它事务）
  • 同时并发执行：多处理器系统中使用，每个处理器处理一个事物（银行开多个窗口）

并发执行导致的问题：

  • 并发势必面临“同读同写”的问题
  • “同读同写”导致破坏事物隔离性和导致数据不一致

什么是数据不一致？

  • 丢失修改：指事务1与事务2从数据库中读入同一数据并修改，事务2的提交结果破坏了事务1提交的结果，导致事务1的修改被丢失了
  • 不可重复性：指事务1读取数据后，事务2执行更新操作，使事务1无法再现前一次读取结果
  • 读“脏”数据（未被最终确定的数据）：指事务1修改某一数据并写回，然后事物2读取该数据。事务1由于某种原因被撤销，数据恢复原值。导致事物1读取的数据为错误数据，也就是脏数据。

什么是锁？为什么需要锁？
因为为了避免多事物并发处理导致数据不一致，所以多事务之间要隔离，事物对某个数据对象操作前，先向系统发出请求，对其加锁。加锁后，事务则对此数据对象具有一定的控制。在释放锁之前，其他事物不能更新此数据对象。

基本锁类型

  • 读锁：又称共享锁。当事物1对某个数据对象加读锁，则其他事物只能对此数据对象加读锁，不能加写锁。直到事务1释放锁（事务2还是可以读取事务1的，但是要真正写入数据库，需要等待事务1执行完成，释放锁才可以）
  • 写锁：又称排他锁（独占锁）。当事物1对某个数据对象加写锁，则其他事物都不能再对此数据对象加任何锁（这就导致事务2只能等事务1的处理完后，才能处理，这样做其实就像串行执行了，效率不高）

悲观锁与乐观锁（本质上是对锁机制一种看待角度）
（写锁）悲观锁：对数据被外界（包括本系统当前的其他事务，以及来自外部系统的事务处理）修改持保守态度，因此，在整个数据处理过程中，将数据处于锁定状态
（读锁）乐观锁：乐观锁假设认为数据般情况下不会造成冲突，所以在数据进行提交更新的时候，才会正式对数据的冲突与否进行检测，如果发现冲突了，则让返回用户错误的信息，让用户决定如何去做。不同的存储引擎支持的锁的类型也是不一样的

存储引擎

  • MYISAM：从ISAM发展而来，mysql4之前所使用的是ISAM，mysql5.5.5之前的默认存储引擎是MYISAM

    • 支持大文件，通常认为是256TB（跟操作系统实现有关）
    • 适合读多写少的应用场景
    • 每个MYISM表的最大索引数据是64，可以通过编译改变
    • BLOB和TEXT类型可以被索引
    • 支持全文索引
    • 仅支持表锁
    • 非事务性引擎
    • .frm文件存储表定义、.MYD数据文件、.MYI索引文件

  • InnoDB:目前mysql的默认存储引擎

    • 事务性引擎
    • 支持行锁和外键
    • 为巨大数据量的最大性能设计
    • .ibdata1自动扩展数据文件，ib_logfiel0/1日志文件

  • MEMORY：将数据存储到内存中的存储引擎

    • 快
    • 不支持BLOB和TEXT列
    • 非事务性引擎
    • 不需要持续性存储，对于读取较频繁的场景

  • NDB：用于Mysql Cluster的集群存储引擎

    • 提供数据层面上的高可用性
    • 例如多表查询等复杂性查询性能相对不是优势
    • 尚在不断完善中

表设计

什么是表？
-正如库是存放表的容器，表是存放数据的一种容器
-表是由行（记录）和列（字段）组成。在数据库中，我们定义“列”的属性。随着数据一条条的增加，“行”不断的增加
-列是结构，行是信息

表的设计上要满足三范式
1.数据库表中的字段都是单一属性的，不可以再分。即能分就分，分到不能再分为止
2.各管各的，别互相影响。别把所有东西放到一个表里
3.决定某个字段值必须是主键（不要重复值）

SQL

SQL是什么？

SQL：Structured Query Language（结构化查询语言）

SQL是一个ANSI标准计算机语言，设计用来访问、操作数据库系统
几乎所有现今的关系型数据库软件（MySQL，MS Access，DB2，informix，MS SQL Server，Oracle，Sybase等等）都使用SQL进行查询，管理及常用操作。

不幸的是，存在着很多不同版本的SQL语言，但是为了与ANSI标准相兼容，它们必须以相似的方式共同地来支持一些主要的关键词（比如:SELECT、UPDATE、DELETE、INSERT、WHERE等等）除了SQL标准之外，大部份SQL数据库程序都拥有它自己的私有扩展。因此，可以说SQL语言是数据库的基础标准语言，各自厂商自行开发的语言置于SQL语言之上。

SQL能做什么？

数据库与表的增、删、改、查、权限管理、视图、索引等等

表格的列与行

数据以表格形式保存

  • 每一列是一个属性，叫列，也叫字段，每一列（属性）以特定数据类型来保存数据
  • 每一行是一条记录，叫行，也叫记录

数据类型

常用数据类型，在创建表格时必须要指定列（字段）的数据类型（不同的类型用于存储不同的数据）

  • 整型 int
  • 浮点型 decimal
  • 固定长度字符串 char
  • 可变长度字符串 varchar
  • 日期 date

SQL语句的分类

Data Definition Language(DDL) 数据定义语言（用于定义数据库与表结构 ）

关键字 功能
CREAT 创建数据库与表对象
DROP 删除数据库与表对象
ALTER 修改数据库与表结构
SHOW 查看数据库与表对象
RENAME 重命名数据库与表对象

Data Manipulation Language(DML) 数据操作语言（操作数据库中表的内容 ）

关键字 功能
INSERT 向一个表格中插入数据
DELETE 删除表格中的数据
UPDATE 更新一个表格中已有的数据

Data Query Language(DQL) 数据操作语言（查询数据库中表内容 ）
关键字|功能
–|–
SELECT|从表格中获取数据

Data Control Language（DCL） 数据控制语言（控制数据库与表的权限）

关键字 功能
GRANT 赋予一个用户对数据库或数据表格等指定权限
REVOKE 删除一个用户对数据库或数据表格等指定权限

Transaction Control(TCL) 事务控制（事务特性）

关键字 功能
COMMIT 提交保存数据操作
SAVEPOINT 为方便rollback标记一个事务点
ROLLBACK 从最后一次COMMIT中恢复到提交前状态
