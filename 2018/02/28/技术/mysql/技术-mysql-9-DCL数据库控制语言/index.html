权限系统

mysql权限系统控制一个用户是否能进行连接，以及连接后能够针对哪些对象进行什么操作

mysql权限控制包含两个阶段：

  1. 检查用户是否能够连接
    连接认证：当连接mysql服务时，mysql通过用户名、密码和主机信息验证是否允许连接 （这里的主机是指允许从哪些主机进行连接,即发起方主机））
  2. 检查用户是否具有所执行动作的权限
    主要指GRAND和REVOKE的赋予权限和撤销权限的动作

授予权限：GRANT

语法

1.权限
操作权限 功能
Create 建立新的数据库或数据表
Drop 删除数据库或数据表
Alter 修改已存在的数据表(例如增加/删除列)和索引
Insert 增加表的记录
Delete 删除表的记录
Update 更新表中已存在的记录
Select 显示/搜索表的记录
INDEX 建立或删除索引
管理权限 功能
file 在MySQL服务器上读写文件
PROCESS 显示或杀死属于其它用户的服务线程
RELOAD 重载访问控制表，刷新日志等
SHUTDOWN 关闭MySQL服务
特别权限 功能
ALL 允许做任何事(和root一样)
USAGE 只允许登录，其它什么也不允许做
2.层级

mysql权限层级

  • 全局层级，指全部数据库
  • 数据库层级
  • 表层级
  • 列层级
  • 子程序层级
层级 格式
特定数据库和表 databasename.tablename
所有数据库和表 *.*
3.用户名

root默认是不可以从远程登录到mysql上的，如果需要root远程连接，需要赋予权限 。但是不建议这样做，因为这样涉及到安全问题。因此还是建议另外建立一个用户帐号来远程连接。

4.发起主机

主机可使用的形式

  • 所有主机：'%'
  • 精确的主机名或IP地址：www.linuxcast.net或192.168.1.1
  • 不精确的主机名或IP使用"*"通配符：*.linuxcast.net
  • 指定一个网段：192.168.1.0/255.255.255.0
