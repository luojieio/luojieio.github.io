细节

  • 所管应用必须是前台运行
  • 所管的应用可以是脚本，因为脚本也是程序，同理脚本也需要是前台运行的，且有执行终端如：#!/bin/bash 或直接/bin/bash command.sh之类
  • supervisor的应用程序配置文件中command中指定方法：如果command没有包含一个前置/，它会在PATH变量中找这个命令，所以在command中应该这么写
    • 方法1.完整的绝对路径
    • 方法2.类似./install.sh
    • 方法3./bin/bash install.sh
  • supervisorctl update或supervisorctl reread + supervisorctl add/remove
  • supervisord关了所管的应用也会关了
  • 如果提示命令无法找到，使用绝对路径来执行，因为它是根据python的PATH来找命令的。
  • 这三个参数不影响执行结果
    • startsecs=5
    • startretries=3
    • autorestart=true
  • 特别注意是当使用脚本时，supervisor托管的是脚本，所以你stop的是脚本，而不是脚本启动的程序，但是当你杀掉程序时，托管的脚本也会停掉，所以它们还是有关联的，只不过你stop托管的脚本时，并不能停止真正的程序这点比较麻烦，而且你有时把脚本当command启动时有问题时，如果报错，建议你去看看是否已经有启动的程序，因为有脚本所带动的程序时，所以，当你stop脚本时，因为程序并没有停止，你再start脚本时，因为原来的程序并没有退出，所以报错起不来，而不是因为supervior不支持脚本。（https://stackoverflow.com/questions/22421140/how-to-set-supervisor-to-run-a-shell-script）
  • 脚本启动的子进程要stop，需要在gate.ini中添加。（https://stackoverflow.com/questions/38650551/supervisor-exec-shell-script-and-stop）
    • killasgroup=true
    • stopasgroup=true
  • supervisorctl中控制程序是以配置文件头中指定的名字来指定，不是文件名
