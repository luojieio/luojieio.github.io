docker-1.基础篇-5.数据管理

发表于 2017-03-15 | 分类于 技术 ， docker |



基于docker 1.12版本

官方参考文档

首先请注意：数据卷与数据卷容器都是用于共享数据而生的

其次请注意：docker镜像是持久化保存的，但是docker容器是寄存在内存中的，当你关闭、重启docker容器，其中的数据也就随之消失了，但主注意数据卷（即重点在于主机挂载数据卷上，这是把主机上的内容挂载上容器中，以供使用，所以它们内容是同步，但是因为是挂载上容器的，所以容器重启、关闭、删除，是不会导致主机挂载数据卷丢失的，但是如果你在容器中修改了挂载的内容，那么它会同步到主机上。一句话概括就是，这东西就类似于远程挂载。因此，在此还是要说一句就是，镜像就像程序，它是永久性的，而容器就像进程，它只是工作时驻留在内存中，当进程关闭或重启后，数据就不见了，因为进程（即容器）会从内存中释放，所以容器中的数据是转瞬即逝的。

用户在使用docker的过程中，往往需要能查看容器内应用产生的数据，或需要把容器内的数据进行备份，甚至多个容器之间进行数据的共享，这必须涉及容器的数据管理操作

如果不挂载，则所有log、用户配置文件都会在docker容器内，如果容器销毁，则所有数据都会丢失。挂载出来方便数据迁移以及管理备份，因为通过挂载可以实现数据的同步，并且由于是挂载形式的，所以哪怕容器被销毁，也仅仅只是把一个远程挂载连接给去掉了，数据本身并不会因为容器被销毁而丢失

容器中管理数据主要有两种方式：

  • 数据卷（data volumes）:用于将本地的目录或文件挂载到容器内的数据卷中，同时以实现容器与主机与容器之间的双向数据卷同步（在容器和主机、容器和容器之间共享数据，并实现数据的备份和恢复）。
  • 数据卷容器（data volume containers）：当需要挂载的数据卷信息很多时，一条条挂载就有点麻烦了，这时我们可以从专门定义一个普通的容器用于专门挂载数据卷的容器，以后别的容器需要挂载信息时，直接引用这个专门的数据卷容器就可以批量的将挂载信息挂载在别的容器上了，而不需要再一条条的挂载，而且它的作用跟数据卷一样，也是双向同步。

数据卷（主机挂载数据卷）

类似于linux下对目录或文件进行的mount挂载操作

数据卷是一个可供容器使用的特殊目录，它绕过文件系统，可以提供很多有用的特性：

  1. 数据卷可以在容器之间共享和重用
  2. 对数据卷的修改会立马生效
  3. 对数据卷的更新，不会影响镜像
  4. 卷会一直存在，直到没有容器使用

利用数据卷容器迁移数据（其实就是利用数据卷的双向同步原理）

备份

将/dbdata上的数据卷备份到/backup目录下打包成backup.tar文件，再利用双向同步原理将/backup目录下的内容同步到本地宿主机上的目录

选项 含义
–name worker 首先利用ubuntu镜像创建一个容器worker
–voluemes-from dbdata 然后让worker容器挂载来自dbdata数据卷容器上的数据卷
-v $(pwd):/backup 挂载本地的当前目录到worker容器的/backup目录

worker容器启动后，使用后接的CMD命令，tar cvf /backup/backup.tar /dbdata 命令来将/dbdata下内容备份为容器内的/backup/backup.tar文件，这样一下，由于宿主机已经通过-v 将本地的目录挂载上去了，所以直接同步到/backup目录上的内容下来，这样本地宿主机的当前目录就有了容器中数据卷

其实这个操作跟我们直接利用数据卷的双向同步的原理一样的，根本不用这么复杂，直接打包好放在数据卷中，然后利用同步原理，自然就同步到本地上了，当然你删除数据卷中的内容，本地的也会删除

而且其实也可以不用将-v 另外单独写出来，直接将-v也写入到数据卷容器中，通过其它容器引用就行了。

所以分解过程：
可以将-v 本地挂载到容器的一步也可以写在数据卷容器中，当然这样写在一起，可能将来有些其它容器引用时，也会将本地目录的挂载信息引用到其它的容器中，而使得其它容器的数据卷内容也同步到本地。只要记得数据卷容器是批量的数据卷所创建的容器就好

然后直接引用数据卷容器，这样挂载信息就都同步到了worker容器，自然也包括了本地目录挂载到容器中目录的挂载信息。

其实执行的命令tar这一步甚至都可以不用加上，由于是同步原理，所以你在本地的目录上以复制也好，移动也好，打包tar也好，都可以将文件备份

因为都是双向同步的。说白了，就是利用双向同步来迁移备份出数据卷内容。

恢复

将本地的存放着backup.tar备份文件的目录挂载到容器的/backup目录，利用双向同步的原理，也就将本地宿主机上的备份文件同步到了容器的/backup目录，再利用解压命令将同步到容器中的backup.tar文件进行解压，也就解压到了/dbdata目录了，也就实现还原了。

当然前提是首先创建一个dbdata2的容器，创建数据卷/dbdata

然后创建另一个新的容器，挂载dbdata2的容器，并使用untar解压备份文件到所挂载的容器卷中

其实说白了备份与恢复就是利用双向同步的原理，即备份就是将备份文件同步到本地宿主机，恢复就是将本地宿主机上的备份同步到容器的数据卷中用于恢复。其实手段不仅可以tar，还可以cp、mv、等等

通过这个双向同步机制，即使容器在运行中出现故障，用户也不必担心数据发生丢失，只需要快速地重新创建容器即可

鉴于它是同步机制，所以推荐定期将主机的本地数据进行备份，或者使用支持容错的存储系统，包括RAID或分布式文件系统，如：Ceph,GPFS，HDFS等。
