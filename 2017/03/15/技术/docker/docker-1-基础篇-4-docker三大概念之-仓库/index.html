仓库（Repository）

其实就是一个个位于注册服务器上的“目录”，类似于代码仓库，是docker集中存放镜像文件的场所

注册服务器是存放仓库的地方，其上往往存放着多个仓库。

每个仓库（目录）集中存放某一类镜像，往往包括多个镜像文件，通过不同的标签（tag）来进行区分。如图：

根据所存储的镜像公开分享与否，docker仓库可以分为公开仓库(public)和私有仓库(private)两种形式

目前最大的公开仓库是Docker Hub https://hub.docker.com （docker的官方默认仓库）
国内的公开仓库有：灵雀云、时速云 、daocloud、 阿里云……

如果不希望公开分享自己的镜像文件，docker也支持用户在本地网络内创建一个只能自己访问的私有仓库。当用户创建了自己的镜像之后，就可以使用push命令将它上传到指定的公有或私有仓库。这样用户下次在另外一台机器上使用该镜像时，只需要将其从仓库上pull下来就可以了。

所以可以看出，docker利用仓库管理镜像的设计理念与git非常相似，这方便了版本的更新与分发，提高了自动化和高效率的维护。这里有点小差别就是git是以git为主master来对应的pull request （合并请求）、push（推向下）、commit（提交上），而docker是以客户端为主master，pull（拉下）、push（推送上），所以有此区别。

自动创建

自动创建（Automated Builds）是基于docker hub实现的
对于需要经常升级镜像内程序来说十分方便。有时候，用户创建了镜像，安装了某个软件，如果软件发布新版本则需要手动更新镜像。而自动创建功能使得用户通过docker hub指定跟踪一个目标网站（目前支持github或bitbucket）上的项目，一旦项目发现新的提交，则自动执行创建。

步骤：

  1. 创建并登录docker hub，以及目标网站，在目标网站中连接账户到docker hub
  2. 在docker hub中配置一个自动创建
  3. 选取一个目标网站中的项目（需要含有Dockerfile）和分支
  4. 指定Dockerfile的位置，并提交创建。之后，可以在docker hub的”自动创建“页面中跟踪每次创建的状态

使用registry镜像创建私有仓库

选项 说明
-d 是指后台守护进程模式
-p 是指将本地主机的5000端口指向容器内的5000端口（切记是本地端口映射到容器端口的顺序）

默认情况下，仓库创建在容器中的目录路径分为两个版本：

  • registry V1版本：/tmp/registry目录下，可以通过-v参数来将镜像文件存放到本地的指定路径上（通过数据卷的方式挂载）以做到持久化保存，因为如果不挂载在本地，当你重启registry存放在/tmp/registry目录下的仓库与镜像就将清除。
    如：将上传的镜像放到/opt/data/registry目录

  • registry V2版本：/var/lib/registry目录下
    如：将上传的镜像放到/opt/data/registry目录

此时，在本地将启动一个私有仓库服务，监听端口为5000，对应存放镜像的本机目录路径为/opt/data/registry

测试：
当使用curl http://192.168.0.100:5000/v2/_catalog能看到json格式的返回值时，说明registry已经运行起来了

registry参考说明：
新版 registry v2对镜像存储格式进行了重新设计，并且和旧版(registry v1)不兼容。
registry v2是由go语言开发，registry v1 是由python语言开发。

docker从1.6版本开始支持registry v2，之前python开发的老版registry在网上已被标为废弃了（没有维护更新，但也可以用）。

由于对旧版是不兼容的，所以之前仓库的数据目录不能直接拿来挂载，需要重新做个新的。

虽然私有仓库已经搭建完成，但是并不代表就可以push和pull操作了，因为从V2版本开始，registry提升了安全机制，通过使用https来替代之前的http进行传输所以需要修改客户机上的docker的配置文件以指定registry地址，以非安全的注册服务器来启动docker进程时强制接受到配置的注册服务器，跳过安全告警。

V2版本的registry提供两种版本的通信：

  • insecure-registry（http形式）
  • secure-registry （官方推荐，https形式，这种需要证书与密钥）

insecure registry

如果采用insecure registry的模式，那么所有与Registry交互的主机上的Docker Daemon都要配置：-–insecure-registry选项

各版本的修改配置不一样：
centos7:

在/etc/sysconfig/docker文件中添加一下配置
ADD_REGISTRY='--insecure-registry 192.168.80.31:5000'

centos6:

在/etc/sysconfig/docker文件中添加一下配置（通过rpm -qc docker-io查看配置文件所在路径）
other_args='--insecure-registry 192.168.80.31:5000'

ubuntu:

在/etc/default/docker文件中添加一下配置
DOCKER_OPTS="--insecure-registry 192.168.80.31:5000'

修改配置文件后，一定要重启docker服务才能生效
centos7:systemctl restart docker
centos6:service docker restart
ubuntu:service docker restart

重启后可以查看进程（ps aux|grep docker ）现进程已经有带上--insecure-registry的参数了

上传前要先给镜像tag一个192.168.80.31:5000/为前缀的名字，这样才能在push的时候存到私库。

这时push就可以上传镜像到所搭建的registry仓库了
