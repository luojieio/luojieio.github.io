docker-2.实战篇-2.apache镜像

发表于 2017-03-15 | 分类于 技术 ， docker |



基于docker 1.12版本

官方参考文档

web服务（web service）和web应用（web app）是目前互联网领域的热门技术

docker来运行：

  • 常用web服务，包括：Apache、Nginx、Tomcat、Weblogic
  • 常用web应用，包括：LAMP、CMS

Apache是目前世界上使用排名第一的Web服务器软件，由于其良好的跨平台、安全性、模块化，Apache被广泛用在多种平台和操作系统上。Apache（阿帕奇）的名字源自美国的西南部一个印第安人部落：阿帕奇族

构建带有Apache服务的docker镜像，请记住，任何docker中的服务程序都要放在前台运行。

docker中运行的程序放在前台运行，这是因为docker本身就是放在后台运行的，当docker内的程序也放在后台运行时，投射到linux底层时就进程就会变成如:httpd&&，这种形式，导致linux无法识别，从而EXIT退出，所以这也就是为什么httpd -D FOREGROUND 来启动apache的原因，为的就是将apache的进程在容器中放在前台来运行，通过docker的封装，放到了linux的后台，从而进程就变成了httpd &这种可被linux识别的形式，从而httpd进程才能成功运行起来

基于ubuntu 16.04 构建

方式一：commit方式构建

步骤：
1.创建并启动基于父镜像的容器

2.安装apache，默认情况下ubuntu的软件源中是不包括apache安装包的，所以需要先更新软件源

3.更新好apt-get源后，开始安装apache

4.安装好后，看看启动程序在哪，再尝试启动一下，发现报了一大堆的缺少变量的错，这些环境变量我们可以在启动容器的时通过-e另外指定。因为在容器里指定，当你commit成镜像，进行run时，这些环境变量会重新读取，导致这些变量依然无效。

当然如果你只想在当前容器进行测试一下apache是否安装成功，测试是否可以正常启动，可以执行一下，来重新生成一次环境变量

5.再尝试启动一下，又报错误了，会提示说缺少指定的运行目录，所以我们根据错误信息创建好目录

6.如果启动还报ServerName的问题，这个可以写在/etc/apache2/apache2.conf配置文件中，也可以在run时，通过-e另外指定变量

7.这些都处理好后，接下来，我们创建一个测试用的页面

8.启动apache，进程已经有了

9.在容器内测试一下，如果没有curl，安装一下curl，页面测试成功

10.接下来，可以exit退出容器，开始commit提交构建镜像了

11.基于新构建的镜像，创建容器并运行，特别需要注意的就是，启动时需要使用-e来专门指定环境变量

12.查看容器情况，端口已成功映射，curl测试成功

总结

apache的启动后面带的-D是指定用于Apache使用的参数，而不是daemon的意思，Apache默认启动就是后台运行的，这一点不能混淆，另外，-D FOREGROUND 是必须同时有的，指传入FOREGROUND参数到Apache进程，即前端运行
