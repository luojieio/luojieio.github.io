官方参考文档

SSH之争议
反方：

  • 容器的思想就是一个容器仅运行一个应用，加SSH违背理念
  • 不需要远程进入容器，直接在宿主机attach、exec连接进入，或通过更复杂的手段执行脚本来完成需要的事务

正方：

  • SSH不耗费资源，所以一个应用和加多一个SSH应用相差不大
  • 有时需要远程SSH进入，而不用每次都从宿主机进入，更方便，更符合linux管理的习惯

可以看到PATH/Dockerfile，默认情况下build是查找路径下的以Dockerfile命名的文件用于构建的。

几个问题在此强调一下：
问题一：Dockerfile文件名是否必须写成Dockerfile首字母大写，可否写成其它名称？
答：可以
如果你所用于build构建而用的配置文件不是Dockerfile命名，需要使用选项-f后接你所定义的文件名，再后接该文件所在的路径，一定不可以反过来，必须是先文件名后路径
如：相对路径的写法

如：绝对路径的写法

问题二：编排顺序验证是否可穿插，如RUN后接ADD再接RUN？而不是一定要全部写RUN，再写ADD，再写其它关键字？
答，可以随意穿插，就像写shell脚本一样，从上至下执行，按你的思路来写即可，没有说关键字一定要写在一起

问题三：是否可以有空行
答：可以。为了Dockerfile文件便于查看，可以留空一行，方便区分不同的段落

问题四：构建过程如果没有构建完成，是否会生成镜像文件？
答：会的，从build构建的原理可知，每运行一次RUN就会commit提交一次镜像文件，所以当你通过Dockerfile文件展开build操作时，如果中途出错，此前commit的镜像文件依然会保存下来
