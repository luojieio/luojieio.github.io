docker-1.基础篇-0.命令选项帮助

发表于 2017-03-14 | 分类于 技术 ， docker |



官方帮助

更新至docker 1.12.3

docker命令行语法格式

  • docker的命令格式与linux工具的命令格式相类似。即：主命令 选项 子命令 选项 。这种形式的命令构成可以随时在后面加上--help来查看具体帮助的。
  • docker COMMAND --help获取各个子命令的更多帮助信息。

docker选项

选项 含义
--config=~/.docker 客户端配置文件的位置
-D, --debug 使用debug（调试）模式。默认为false。可以指定-D=true或false
-H, --host=[unix:///var/run/docker.sock]，tcp://[host:port] 在daemon（后台守护进程模式）下绑定的socket，通过一个或多个tcp://host:port，unix:///path/to/socket，fd://* 或 fd://socketfd来指定
-h, --help 打印帮助
-l, --log-level=info 设置日志记录级别
--tls 使用TLS; 暗示--tlsverify
--tlscacert=~/.docker/ca.pem 信任证书仅由本CA签署
--tlscert=~/.docker/cert.pem TLS证书文件的路径
--tlskey=~/.docker/key.pem TLS密钥文件的路径
--tlsverify 使用TLS并验证远程
-v, --version 输出版本信息并退出。也可以-V=true或false来指定是否开启。默认为false。
--api-enable-cors=true或false 在远端API中启用CORS头，默认为false。（旧版本）
-b="" 将容器挂载到一个已存在的网桥上。指定为'none'时则禁用容器的网络（旧版本）
--bip="" 让动态创建的docker0采用给定的CIDR地址，与-b选项互斥（旧版本）
-d=true或false 使用daemon模式，默认为false（旧版本）
--dns="" 让docker使用给定的DNS服务器（旧版本）
-g="" 指定docker运行时的root路径。默认为/var/lib/docker （旧版本）
·–icc=true或false· 启用容器间通信，默认为true（旧版本）
--ip="" 绑定端口时的默认IP地址，默认为0.0.0.0（旧版本）
--iptables=true或false 禁止docker添加iptables规则，默认为true（旧版本）
--mtu=VALUE 指定容器网络的mtu。默认为1500（旧版本）
-p="" 指定daemon的PID文件路径。默认为/var/run/docker.pid（旧版本）
--registry-mirror=:// 指定一个注册服务器的镜像地址（旧版本）
-s="" 强制docker运行时使用给定的存储驱动（旧版本）
--selinux-enabled=true或false 启用SELinux支持。默认值为false。SELinux目前不支持BTRFS存储驱动（旧版本）

docker子命令

子命令 含义
attach 依附到一个正在运行的容器中
build 基于 Dockerfile构建一个镜像
commit 基于一个修改的容器创建一个新的镜像
cp 从容器中复制文件或目录到宿主系统中
create 创建一个新的容器
diff 检查一个容器文件系统的修改
events 从服务器上获取实时事件
exec 在一个运行中的容器中执行一条命令
export 将容器内容导出为一个数据流形式的tar包归档
history 显示一个镜像的历史信息
images 列出本地的所有镜像
import 从tarball包中导入作为创建一个新的文件系统镜像
info 显示整个系统的信息
inspect 显示容器或镜像的底层详细信息
kill 杀死一个正在运行中的容器（包括进程和所有相关资源）
load 从一个tar包归档中载入一个镜像
login 注册或登录到一个dokcer的注册服务器
logout 从docker的注册服务器登出
logs 获取容器的日志
network 管理docker网络（新加入）
node 管理dockerswarm节点（新加入）
pause 暂停容器里的所有进程
port 查看对外提供的端口
ps 列出容器
pull 从docker注册服务拉取一个镜像或仓库
push 推送一个镜像或仓库到docker注册服务器
rename 重命名一个已经存在的容器
restart 重启一个运行中的容器
rm 移除一个或多个容器
rmi 移除一个或多个镜像
run 创建一个新的容器，并在其中运行给定命令。
save 保存一个镜像到一个tar包
search 从docker hub中搜索镜像（在docker index中搜索镜像）
service 管理docker服务（新加入）
start 启动一个处于停止状态的容器
stats 显示一个容器的资源使用数据流情况
stop 停止一个运行中的容器
swarm 管理docker swarm
tag 为镜像打标签
top 查找一个容器中运行的进程
unpause 不暂停一个暂停中的容器
update 更新一个或多个容器的配置（新加入）
version 显示docker版本信息
volume 管理docker数据卷（新加入）
wait 阻塞,直到一个容器停止,然后打印它的退出代码
