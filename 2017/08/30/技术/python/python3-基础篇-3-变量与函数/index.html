变量

变量说明

变量，顾名思义，值可以变化，即你可以使用变量存储任何东西，变量只是你的计算机中存储信息的一部分内存。通过内置函数id(变量名)可以来查看变量在内存中的存储空间地址编码，注意这个编码是python中的，而不是真正的内存中的，因为python是高级语言，不处理底层。
（对象，在很多其他编程语言中叫做“变量”，是一个基本的计算机术语，但我们说对象和变量其实质虽然相同，但含义是不同的，对象是相对于类来说的，具有更高一层的含义）

  • 值是一段代码需要处理和运行的最小单位，它可能是一个数字，或者一串字母。在实际生活中，当我们说“值”的时候，潜意识中就自动对它进行了分类。比如：数字、字母、单词 等等。计算机在处理数据的时候也是根据具体的数据“类型”来获取不同的“属性”，采用不同的处理“方法”的。
  • 变量是程序中最为基本的内容之一，变量名就是一个变量内容的名字（另一个重要的内容是“函数”）
  • 通过等号，我们把变量（名）指向一个值（即变量内容）

比如：
myString="hello python"，其实际上就是我们把“hello python”这句话赋予给了myString这个变量，或myString这个变量名指向了“hello python”这个变量内容

即：
variable = value

命名要求

  • 它可以由字符、数字、下划线（_）组成，但是它不能用数字作为开头，即首字母必须是大小写字母或下划线
  • 不能有空格，空格是作为命令的分隔，容易导致错误。横杆-，因为横杆是减号占用了
  • 对大小写敏感
  • 简单明了，变量要有意义，尽量用名词，不可以使用关键字，这样做会导致将原来的关键字的功能被覆盖（python支持中文作为变量名，但是不建议这样做，而且不要写拼音，这样太Low了，写英文是最通用化的）

书写规则：驼峰规则

一般有三种命名规则：（全大写、全小写不便于识别）

  • 小驼峰式命名法（lower camel case）：第一个单词以小写字母开始，第二个单词的首字母大写，例如：myName、aDog
  • 大驼峰式命名法（upper camel case）：每一个单词的首字母都采用大写字母，例如：FirstName、LastName
  • 以下划线“_”来连接所有的单词，例如：my_name、last_name

函数

函数说明

所谓的函数就是可以实现一个特定功能的代码块，这是编程的的另一个基本单元（先前一个基本单元是变量），它有自己名字，用来唯一的标识这个函数。当我们需要实现某个功能的时候，我们会“召唤”函数来帮我们实现这样的功能。
“召唤”的方法就是：函数名+()来调用这个函数，有时候还会在()中传进来实现这个功能需要使用的变量参数

为什么使用函数？

  1. 降低编程的难度
    通常将一个复杂的大问题分解成一系列更简单的小问题，然后将小问题继续划分成更小的问题，当问题细化为足够简单时，我们就可以分而治之。这时，我们可以使用函数来处理特定的问题，各个小问题解决了，大问题也就迎刃而解了。
  2. 代码重用
    我们定义的函数可以在一个程序的多个位置使用，也可以用于多个程序。此外，我们还可以把函数放到一个模块中供其他程序使用。同时我们也可以使用其他程序员定义的函数，这样就避免了重复劳动，提高了工作效率。
  3. 一致性
    一处修改处处生效
  4. 可扩展性
    想要增加功能，直接修改函数体即可

函数的形式参数与实际参数

在定义函数时，函数名后面圆括号中的变量名称叫做”形式参数“，或简称为”形参“
在调用函数时，函数名后面圆括号中的变量名称叫做”实际参数“，或简称为”实参“

切记：在以位置传参时，实参跟形参位置和数量一定是一一对应上的，不可以多，也不可以少，位置也一样（当然我们可以通过默认参数及参数组的方式来实现少传或多传的问题）。但需要清楚的是，形参是用于定义函数，实参是实际使用函数，所以两者的命名并不是一定要一致，它们没啥关联性。只要数量与位置对应上即可。

内建函数

绝大多数函数是我们自己写的，但是日常工作中有一些函数实现的功能是非常基本的，并且大家对结果的要求都一致，这些函数就预先被写好允许大家进行直接调用，我们称之为“内建函数”。

范围函数range()

range()：函数会生成一个数字列表，具体列表依据传入的参数不同而定。range意为范围的意思。

格式：（其实你只要记得[)的规则就没问题了）

  • range(n)：会生成一个0，1，2，3，4…n-1的列表
  • range(m,n)： 会生成一个m,m+1,m+2,…n-1的列表
  • range(m,n,k)： 会生成一个m,m+k,m+2k,…n-1的列表

range()函数经常和for循环一起搭配，for通过range()函数生成的数字来计数，共同形成循环

函数的变量适用域

说明：
在编写函数的过程中，我们可能根据需要在函数内部声明一些变量。这些变量的寿命非常短，函数在使用它们的时候非常残忍，“狡兔死，走狗烹”，用完就毁灭它们是这些变量的命运。然而在外部本来就存在的变量不会遭此厄运。

解释：
运行一下，可以看到，在函数运行后，在函数外本来就存在的dog,cat依然存在。但是在函数内部产生的变量：animal，虽然在函数运行的时候产生并存在，但是当函数执行完之后就死亡了，解释器会产生：NameError: name ‘animal’ is not defined，这样的错误。这样的设定并不意味着残忍，而是一般在函数运行中产生的变量，一般只会在函数内部产生作用（即局部作用），函数外面定义的变量会对全局产生作用（即全局作用）。而对于函数内的变量来说，如果让它一直存在下去，会是对资源极大的浪费，如果在函数之外你会多次用到animal这个变量，正确的做法是：把animal=dog+cat从函数中提出来，放在更“广阔”的域中。即写成函数之外的全局变量或强制在函数体内声明为全局变量（有调用才生效）
注意：哪怕局部变量的变量名跟全局变量的变量名相同，它们也是两个不同意义的变量，作用范围不同。

也可以强制声明局部变量为全局变量，但前提是该函数有被调用，否则仅仅只是定义的函数中有global因为没有被调用，所以也就没有实际产生作用
如：只要加上global关键字即可

全局变量与局部变量总结:

  • 全局变量与局部变量同名时，局部变量的优先级优先于全部变量
  • 局部变量可以变为全局变量通过global 变量名 的方式来声明，但不建议这样秀操作，不利于排错
  • 局部变量内修改变量时，要看这个类型是否是可变类型还是不可变类型。可变类型，比如列表、字典…这些是可以修改到全局变量的，但不可变类型，比如字符串、数字…这些就不可以修改了。其实从内存地址空间上看都没有修改。

函数的返回值

  • 函数被调用后会返回一个指定的值，这个值可以是由函数默认提供的None，也可以是我们自己定义的
  • 自已定义函数的返回值使用 return 返回值来实现，return的类型没有限制，可以是任意类型，比如数字，字符串，列表，字典等等，甚至是函数，因为在python中任意东西都是对象
  • return的个数没有限制，可以一个也可以是多个
  • 特别强调的是return执行后，函数终止

返回值数=0：返回None
返回值数=1：返回object，即你定义的单个对象是什么就返回单个什么
返回值数>1：返回tuple，即当你定义的返回值很多时，就将其包裹在一个元组内，作为单个元组整体返回

为什么要有返回值？
我想要知道函数的执行结果，因为后面的程序逻辑需要根据这个执行结果来判断执行不同的代码来做出不同的操作

区分返回值和打印

重点1：返回值赋予变量

重点2：return执行后，函数终止

总结return与print的区别：

  1. return 可以赋值给变量，print不行（这点来说shell的echo可以同时实现这两个功能）
  2. return 是不显示在屏幕的（交互模式下显示），print会打印到屏幕上。（可以想像一下shell中的echo $?的返回值的概念）
  3. return 执行，函数就终止，print 不会终止函数，只是打印输出而已

建议多使用return，而少用print，除非你在测试时确实要将结果打印到屏幕

函数的多类型传值、冗余传值

多类型传值（也叫参数组）

  • 将列表、元组、字典中的”元素“对应上函数的形参位置来传值，而不是简单的把列表、元组、字典一整个整体作为一个形参来传值
  • 多类型传值时，定义函数和定义列表、元组、字典并没什么不同，只是使用函数时，列表、元组加*，字典加**而已，字典还有个小细节就是定义的函数中的形参要与字典的关键字key相对应
  • *args： *表示多个的意思，args是”变量“参数的意思
  • **kwords： **表示多个的意思，kwords是”关键字“参数的意思
传列表：fun(*args)
传元组：fun(*args)
传字典：fun(**kwords)

上面说的是多类型传值，将列表、元组、字典这些容器里的元素一一对应上函数的形参

另外，再加上有默认参数的存在，所以现在可实现：

  • 不传值–>可以直接使用默认参数
  • 传单个值–>可以按位置传参或关键字传参直接指定单个实参，另一个则可以使用默认参数
  • 传完整值–>完全按形参需要的值给予，一一对应的情况时，无论是直接传值、变量传值（或交互式）、或是多类型传值
  • 传超出值–>冗余传值（接下来说）
