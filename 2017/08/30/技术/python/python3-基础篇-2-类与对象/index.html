面向过程与面向对象

在谈“类”之前我们先要知道什么是“面向过程”与“面向对象”的编程？
形象说法1：

  • 面向过程：是编年体，以时间为主线，记录着某年发生了某事，它是单线性的
  • 面向对象：是纪传体，以人为主线，记录着某人做了某事，它是多向性的

形象说法2：

  • 面向过程：就是分析出解决问题所需要的步骤，然后用函数把这些步骤一步一步实现，使用的时候一个一个依次调用就可以了。
  • 面向对象：是把构成问题事务分解成各个对象，（就像把原来的函数进行分类了），建立对象的目的不是为了完成一个步骤，而是为了描叙某个事物在整个解决问题的步骤中的行为。

例如五子棋
面向过程的设计思路就是首先分析问题的步骤：
1、开始游戏，2、黑子先走，3、绘制画面，4、判断输赢，5、轮到白子，6、绘制画面，7、判断输赢，8、返回步骤2，9、输出最后结果。
把上面每个步骤用分别的函数来实现，问题就解决了。

面向对象的设计则是从另外的思路来解决问题。整个五子棋可以分为：
1、黑白双方，这两方的行为是一模一样的
2、棋盘系统，负责绘制画面
3、规则系统，负责判定诸如犯规、输赢等。
第一类对象（玩家对象）负责接受用户输入，并告知第二类对象（棋盘对象）棋子布局的变化，棋盘对象接收到了棋子的变化就要负责在屏幕上面显示出这种变化，同时利用第三类对象（规则系统）来对棋局进行判定。这种通过不同对象之间的API调用来完成问题。

类（类型）

面向对象编程，主要核心就在于分“类”，就像按某种特性分“组”一样。一种类型中，包含这种类型的“属性”与“方法”，我们要使用时，直接调用这个类型的“属性”与“方法”的API即可。不同的数据，需要定义不同的数据类型，以便存储不同的数据。这就像喝茶要用杯，吃饭要用碗，装水要用桶一样。查看数据类型可以使用type()这个函数。

类型都是用来存放数据的容器，其中的数据以一种特定的形式存放在这些由小及大的容器中，在用户需要的时候通过API调用的方式拿出来即可。这些容器就像由小及大的水桶，数据就像水，不同容量的水（数据）用不同的水桶（类型）来装，就像不同类型的数据，用不同的类型来盛放一样。其实从广义上说它们都是数据结构，狭义上说的数据结构是存储大量数据的结构：列表、元组、字典、集合。

字符编码

python3的默认字符编码为Unicode，所以python3天生就支持中文，而python2的默认字符编码为ASCII，所以需要另行在开头声明或使用Unicode字符串，即需要在字符串前加上前缀u或U。除了多了一个u之外，与普通字符串没啥区别，转义字符和多行表示法仍然有效。例如，u"This is a Unicode string."

如果中文字符串在python3环境下遇到 Unicode Decode Error（即Unicode解码错误）这是因为.py文件保存的格式有问题。因为python3程序默认使用ASCII码，所以使用中文时会报错，因此你的脚本程序使用中文时需要先声明使用utf8的方式存储

字符编码 说明
ASCII 一个字符用1字节表示，仅支持纯英文。
ASCII编码，最早的计算机在设计时采用8个位（bit）作为一个字节（byte），所以，一个字节能表示的最大的整数就是255（二进制11111111=十进制255）。
0 - 255被用来表示大小写英文字母、数字和一些符号，这个编码表被称为ASCII编码，比如大写字母 A 的编码是65，小写字母 z 的编码是122。当初也只用了127个，预留了128-255区间作为扩展。
但由于中文字数量多，剩余的区间不够用于存放中文字，于是在扩展中取一位用于标记为一个索引，然后从这个索引中扩展出一个7445个汉字和其它符号组成的编码表格。这就是1980年诞生的GB2312字符编码集。
但由于GB2312支持的汉字太少了，1995年汉字扩展到21886个符号，分为汉字区和图形符号区，这就是GBK字符编码集，GBK是兼容GB2312字符编码集的。
后续又扩展到GB18030字符编码集，这些编码都是向下兼容的，也就是说GB18030都是包含GBK/GB2312/ASCII字符编码的。
发展过程：
ASCII–>GB2312–>GBK–>GB18030
每个国家都用这种方法搞一套，就导致会有非常多混乱的版本，于是国际标准组织产生了unicode编码
Unicode 一个字符用2字节表示，Unicode把所有语言都统一到一套编码里，可以支持全部语言，这样就不会再有乱码问题了，但是会造成浪费，因为这相当于原来占用1个字节的，现在需要占用2个字节的空间。原有的英文编码从单字节变成双字节，这就需要把高字节全部填为0。
UTF-8 一个字符用3字节表示（可变长度，英文还是按1字节，中文用3字节），UTF-8是Unicode的一个扩展集，其本身就是基于Unicode的

字符编码的原理：
如果要表示中文，显然一个字节是不够的，至少需要两个字节，而且还不能和ASCII编码冲突，所以，中国制定了GB2312编码，用来把中文编进去。类似的，日文和韩文等其他非英语语言也有这个问题。中国利用ASCII编码，通过一一对应的字对应于相应的九宫格图片，再将此图片对应于ASCII码的方式，发明出如gb2312这种字符编码

  • 英文字母与符号–>图片–>ASCII编码（十进制）–>二进制
  • 中文–>图片–>英文字母与符号–>图片–>ASCII编码（十进制）–>二进制
    因为是1个字节的限制所以需要做一个中间转换，但是当出现unicode后，就不需要这种转换了，容量足够大，可以容纳中文字符

字符转换过程：
ASCII<–>Unicode<–>utf-8

  • 内存中使用unicode ，因为不可以同时存在ASCII和unicode，这样会造成错乱，但是如果只是ASCII又会造成无法处理中文，因此当我们需要使用内存处理时使用unicode来处理，当需要存储到硬盘时再转换成utf-8
  • 硬盘中使用utf-8

列表（属于序列范畴，数据结构，用[]括号）

说明：
任何语言中都少不了”数组”，数组是一种结构，用来存储一组数据，在python3中他们是以列表（list）和元组（tuple）的形式存在的，更高级的是：列表和元组可以存储任何类型的对象。

列表具有的特征：

  • 列表的开始和结束要加上中括号[]
  • 采用逗号将列表的项与项分开
  • 使用=号操作符将整个列表赋予一个变量
  • 列表中可以放置任意的数据类型，如果是字符串，需要打上引号（字符可以是单引号也可以是双引号）
  • 列表中的每一个元素都是可变的（意味着我们可以添加、删除、修改）
  • 列表中的元素是有序的，也就是说每一个元素都有一个位置（也就是说它是序列，可以通过索引号来取值）
  • 列表可以容纳python3中的任何对象（看下面的比如就知道）
  • 可以定义空列表，list=[]
  • 定义单一元素列表：list=[1] 不用像元组需要在末尾加上逗号,

列表的不足：

  • 严重依赖于位置来取值，位置一变动会导致取值错误，所以元素的位置不能动
  • 如果数据量很大的话，对于记住元素的索引位置是困难的

可变类型与不可变类型

元组与列表类似，只不过元组不能修改。可能你会觉得这么“死板”的元组，究竟有什么用处呢？这里我们得先从“不变性”来说起，一个数据不可变到底意味着什么？
数据的不变意味着：一旦这个数据被创建了，他的值就不能再被更新，除非放弃它，重新创建一个新的对象。

不可变性不是一件坏事，比如在某些时候对方的程序需要使用我们的数据实现某种功能，然而我们不希望对方将此数据进行任何的修改。那么，如果我传来的数据是一种不能被随意篡改的类型，这无疑是最好的解决方法。

不可变的类型有三种：

  • 数字
  • 字符串
  • 元组

其中数字、字符串都是标量类型，而元组是一个容器，它其中可以放入很多类型的元素。而这个容器，“只可远观不可亵玩焉”。因为一个元组不能被修改，那么”创、删（指删除整个元组）、增、减（指删减元素）、改、查”这些操作我们能做哪些呢？显然增加、减少、修改操作是不能进行的。只有创建、删除和查看操作可以。

元组的特性：

  • 定义一个空元组：tuple=()
  • 定义单一元素元组：tuple=(2,) 要记得加逗号,否则不是元组类型

格式：tuple_name=（item1,item2,item3）

字典（属于映射范畴，数据结构，用{}括号）

说明：为什么说是映射不是序列，因为它不分顺序，不是通过索引号来取值。如果有其他语言背景的同学肯定知道关联数组或哈希表。在python3中他们是以“字典”的形式存在的，表现形式是：键-值 （key-value）

字典的特征总结如下：

  • 字典中的数据必须是以键值对的形式出现
  • 逻辑上讲，键是不能重复的，而值可以重复
  • 字典中的键（key）是不可变的，也就是无法修改的，而值（value）是可变的，可修改的，可以是任何对象（因此你不可以以列表这种可变值作为key）
  • 键与值不能脱离对方而存在，也就是说如果你定义了一个键，就必须有一个相对应的值，它们必须是一一对应的。
  • 字典是无序的，因为有key,所以不需要索引号（下标）
  • 字典是python3中唯一的映射类型（哈希表），因为序列是有序的，哈希表是无序的
  • 字典的对象是可变的，但是字典的键必须使用不可变对象，并且一个字典可以使用不同类型的键值
  • 几乎所有类型的对象都可以作为“键”，不过一般还是以数字或字符串最为常用，key也可以是变量，使用变量作为key之前这个变量要事先已经定义好的。另外，使用变量做为Key时，在取值时，可以是变量名也可以是变量所对应的值，它们都可以映射到字典中的value
  • 所有类型的对象都可以作为“值”，甚至是一个列表，每个“键-值”的搭配都是用逗号“,”来隔开
  • 字典要用“{}”花括号来包裹
  • 注意引号一定不能是中文的
  • 字典的多级嵌套跟列表和元组的使用方式类似，也是[key1][key2]这样一层一层的往下剥
  • 切片是只针对序列使用的，字典是映射，不是序列，所以字典是不能切片的

字典的好处：

  • 相对于列表来说，字典的取值是以key-value的键值对的形式，不依赖于位置，所以元素的位置变动也没关系
  • 对于数据量大的情况时，不需要去记住元素的具体位置，直接以人类更容易理解的key来映射出对应的值

格式：{键1:值1,键2:值2,键3:值3,,,,,,}
注意：

  • 注意键/值对用冒号分割，而各个对用逗号分割，所有这些都包括在花括号中。
  • 键必须是唯一的，就像如果有两个人恰巧同名的话，你无法找到正常的信息
  • 只能使用不可变的对象（比如数字、字符串）来作为字典的键，但是你可以将不可变或可变的对象作为字典的值，基本说来就是，你应该只使用简单的对象作为键
  • 字典中的键/值对是没有顺序的，如果你想要一个特定的顺序，那么你应该在使用前自己对它们进行排序

在生活中我们会发现很多数据是“键-值”来组织的，比如一个家庭结构中：

大总结

综上类型总结： 数字、字符串、列表、元组、字典、集合

  • 序列：字符串、列表、元组 （因为通过序列号可以索引进行单一取值和多取值（拼接或切片））
  • 数组（数据结构）：列表、元组、字典、集合
  • 映射：字典（因为通过Key：value 可以映射对应的值，而不是通过索引号来取值）
  • 不可变类型：数字、字符串、元组 （通过id()可以判断，会新生成一块内存地址空间，而不是说在原内存地址空间修改，所以这三个是不可变类型的数据）
  • 可变类型：列表、字典、集合（通过id()可以判断，还是使用原来的内存地址空间，没有重新生成一块新的内存地址存储空间，所以是可变类型的数据）

None

空值类型与0、空格不太一样，None是没有值存在的空值类型

为什么一个对象会是None?
这是因为当你在定义函数的时候没有写return返回值时，当你调用函数时，企图把这个根本不存在的“返回值”储存在一个变量中时，变量实际的赋值结果将会是None，这也是默认的值。
因此我们可以了解到的是python3跟shell不同，shell的echo时，可以将“打印”与“返回”集成在一起，而python3则是将其两者绝对的分开，print是只打印，return是只返回值
如果你只写了print，则当你要将结果赋值给变量时，因为没写return，所以不能将想要打印的结果赋值给变量，而是将默认的None赋值给变量。当然，当你想要设定一个变量，但又没想好它应该等于什么值时，你就可以这样：a_thing=None

定义类

我们使用class关键字来定义一个类，就如同创建函数时使用的def定义一个函数一样，不过需要特别注意的是class定义类不需要在后面带上()，而def定义函数时需要带上()，因为函数需要传参。

如：

提示：类中的属性与方法，跟非类中的变量与函数一样，只是叫法不同而已，之所以有这样的叫法上的区别也是为了区别于类与非类的调用形式。

重点：类的方法与普通的函数只有一个特别的区别，方法必须有一个额外的第一参数名称self，但是在调用这个方法的时候你不为这个形参赋值。这个特别的形参指对象本身。

定义对象（类的实例化）

如：

引用属性与方法

print(coke().formula)     #引用类的属性
coke().drink()          #引用类的方法

print(a.formula)           #引用实例（对象）的属性
a.drink()               #引用实例（对象）的方法

魔术方法

init() 用于初始化，我们在创建实例的时候不去引用init()方法，其中的命令也会先被自动执行
dict 默认隐藏属性，是一个字典，存储着类与实例的属性

类的继承

  • 父类：公共通用的，将通用性的东西写在父类中
  • 子类：私有独立的，将差异性的东西单独写在子类中
  • 在子类中的定义中直接在()中把父类当成参数写入，这样就继承了父类的东西
  • 类中的变动会影响到对象，对象中的变动不会影响到类

在python中属性的引用机制是自外而内的，也就是说先对象，后类，当你创建了一个实例之后，准备开始引用属性，这时编译器会先搜索该实例是否拥有该属性，如果有，则引用，如果没有，则将搜索这个实例所属的类是否有这个属性，如果有，则引用，没有那就只能报错了。因此，正是在实例化对象的过程中，默认会先初始化引用__init__这个魔术方法的，所以初始化后生成的这个对象默认就有了这个__init__的属性，所以对象的属性中如果对象与类中的属性同名时，默认情况下init会更优先，也就是基于这个自外而内的机制原因。

重点总结：
定义类时：（时刻想着有个self，这是与非类的函数唯一的区别）

  • 属性:var
  • 方法:method(self)

引用类时：（当这个self是透明的不存在的，忘记它的存在）

  • 属性:obj.var
  • 方法:obj.method()

对象（具体实例）

对象是针对于类而言的，python使用类型来存放不同的数据，任何类型都可以实例化为具体的对象，实例化的过程就是定义变量的过程。
对象如同变量名，即定义变量的过程就是将“类”实例化为“对象”的过程，当然不同类型实例为对象的方式不同，但总体来看都类似于定义变量。

比如：
字符串类型实例化为对象：str_obj='this is string'
列表类型实例化为对象：list_obj=[1,2,3,4]
……

我们在上面学到的类型都是一些简单的基础类型，如果基础类型无法满足我们的需求，我们还可以自定义类型。

python的对象都拥有三个特性：名字、类型、值

  • “名字”：是它的唯一的身份标识自己，所以这个名字应该在“一定范围内”具有唯一性（就是变量名嘛）
  • “类型”：决定了这个对象可以保存什么样类型的值，可以进行什么样的操作（python是动态语言，所以不需要再另外声明类型）
  • “值”：表示了对象的数据项（就是变量的值嘛）

比如：i=10
这里就声明了一个对象（变量），它的名字叫i，是一个int（整数），它是值是10，在声明一个对象的时候，我们使用等号=

值得注意的是：在这里并没有显式地写出它是一个int（整数），当你给他赋值的时候，计算机会自动识别出来它是一个int。如果是C这样的静态语言，应该声明为int i

对象你可以理解为针对类的使用时，具体操作的某一个实例化的目标体，比如文件file这个类型，当它实例化一个对象f时，当我操作f这个目标时，就是在操作f所对应的文件的操作，因为在实例化前有open(filename，r+)，这样当你操作f时就是在操作filename这个文件。
