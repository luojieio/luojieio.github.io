python的历史

由Guido van Rossum （荷兰人） 在1989年在阿姆斯特丹为了打发无聊的圣诞节而开发的“高级、动态、解释型、面向对象的第四代编程语言”,初衷是为了辅助ABC这个语言实现相关功能的一个插件，结果无意发现开发出了一门语言。
python名字的由来是因为 Guido 本人是天空马戏团的粉丝，所以取了“monty python”中的python作为这个语言的名称。

python的发展阶段：

  • CNRI时期：CNRI是资助Python发展初期的重要单位，Python 1.5 版之前的成果大部分都在此时期内完成
  • BeOpen时期：Guido与BeOpen公司合作，此一期间将python 2.0 推出，甚至python 1.6 也同时问世，但原则上已经分别维护了（之所以要分别维护是因为两个版本互不兼容）
  • DC时期：Guido已离开BeOpen公司，将开发团队带到了Digital Creations （DC）公司，该公司以发展zope系统闻名，因此这项合作也颇受关注
  • Google时期：Python 3.0

python的特点

  • 简单易学不用过度关注语言本身的语法
  • 集解释性与编译性于一身
  • 面向对象、面向过程
  • 高级语言，无需考虑诸如管理内存一类等底层
  • 可扩展性及可嵌入性（比如：你可以把部分程序用C或C++编写，然后在Python程序中使用它们。与此相反，可以把Python嵌入C/C++程序，提供脚本功能）
  • 免费、开源
  • 可移植性强
  • 丰富的库帮助你处理各种工作，直接用即可，而不用从头开始写（除了python安装自带的标准库外，还有很多第三方库，如：wxPython、Twisted、Python图像库等等）

python的应用领域

python适合的领域：

  • web开发（比如：Django/pyramid/tornado/bottle/flask/webpy）
  • 网络服务（比如：twisted/requests/scrapy/paramiko）
    • 系统工具和脚本
    • 爬虫
  • 科学计算（比如：scipy/pandas/python xy）
  • gui图形开发（比如：wxpython/pyqt/kivy）
  • 运维自动化（比如：openstack/saltstack/ansible/腾讯蓝鲸）
  • 作为”胶水“语言把其他语言开发的模块包装起来方便使用

python不适合的领域：

  • 贴近硬件的代码（首选C）
  • 移动开发：IOS/Android有各自的开发语言（Object-C，Swift/Java），还有现在的移动开发使用HTML5
  • 游戏开发：C/C++（因为需要渲染，所以对处理效率要求高）

python和其他语言的对比

开发语言 类型 运行速度 代码量
C 编译为机器码 非常快 非常多
Java 编译为字节码 快 多
python 解释执行，每次执行就进行编译 慢 少

区别：
C/JAVA是将源码编译的。就类似于学会了一门语言，直接来跟机器对话。
python是边执行边编译，就类似于中间的一个翻译，（当然python也可以进行编译）另外python源码不能加密。
python有一个python的虚拟机做为中介，边解释边执行，这点类似于java的jvm，也正是因为这样，所以它的兼容性很好，因为它是跑在内置虚拟机上的

python版本

python当前有两个大版本：2.7与3.3
这两个大版本之间相互有点不兼容，当前主要的Python的第三方库是基于2.7版本的，3.3版本早于2.7版本发布，这是因为3.3版本的不兼容，所以开发了一个2.7的过渡版本用于逐渐过渡到3.0系的大版本。后续将主要是3.3版本。
查看python版本：python -V

python 2.X vs python 3.X：

  • python 2.X是遗产，python 3.X是现在和未来的语言。python 2.7是2.X系最后的版本了，不会再有重大的更新了。python 2.X 仅维护到2020年,后续的python 3.X 将不再考虑向下兼容python 2.X
  • pythonn 3.X 全是unicode ，完全支持中文，python 2.X 还是ASCII码
  • python 3.X 中的print是函数，python 2.X是命令
  • 模块名的规范
  • 1/2终于等于0.5
  • raw_input没了，全变成了input
  • class Foo:写法不能用了，只能class Foo(object)

twisted 目前还是仅支持2.7版本(http://twistedmatrix.com/trac/milestone/Python-3.x)

获取帮助

print help(int) 或 help(int)

如果直接在python的交互模式下help(int)，也可以看到显示结果，但是这样是以属性的方式显示，即连带它是如果实现的都显示出来了。而不像打印print这种不将实现原理打印出来，而仅仅只是结果显示出来。

注意：如果你想获取比如print的帮助，需要加上’’引号，那样print就可以理解我是希望获取关于print的帮助而不是想要它打印东西，正确写法：help('print')

python代码风格

  1. 从上到下执行，所以变量也是从上到下，不像C可以在编译前先将变量导入，所以无论变量写在哪都可以。而python中的变量位置不可以随意写的，否则出来的效果可能就不一样。python在执行操作前只是检查语法是否正确，而没有导入变量
  2. 区分大小写
  3. 跟shell同样的运行方式，交互式和源代码文件式。执行方式也相同，shell脚本是sh或bash之类的，python脚本是python打头的。如果写了#!则可以直接以相对或绝对路径来执行（注意先赋予执行权限）。
  4. 缩进要统一，因为python没有像shell的do done之类的东西作为起始符与结束符，而是通过缩进的层级关系来切分代码的关系。根据缩进格式来确定哪些代码是需要结束的。所以严格管理缩进，左边一定要顶格写。同级的一定要保持对应，不同级的一定要缩进。
  5. python规定必须使用4或2个空格或tab制表符来表示每级缩进（但是在实际编写中可以自定义空格数，但是要满足每级缩进间空格数相等，即使用了tab就全部都以tab来缩进）。随着缩进深度的增加，代码的层次也在加深，没有缩进的代码块是最高层次的。使用缩进对齐这种方式组织代码，不仅代码风格优雅，也大大提高了代码的可读性。这样一来，你看到的就是一个个优美的代码块，而不是令人费解的“字符流”
  6. 在真正开始执行解释之前,python会对你的代码进行一次全面的浏览一遍，看看是否有语法和缩进和变量定义的错误，才会开始展开解释过程

什么需要通过缩进来书写代码？
因为python没有结束符的原因，所以必须通过缩进层次结构来确认哪些代码块隶属于哪一层次，从而执行相应的代码块并结束。另外最上层的代码必须顶格写也是这个原因

注释

所谓注释，就是写在代码中用于解释下面代码是什么意思的内容。注释是用来说明代码的，给自己或别人看，而程序运行的时候，Python解释器会直接忽略掉注释，所以，有没有注释不影响程序的执行结果，但是影响到别人能不能看懂你的代码。

python中的注释使用#号进行标记，从#开始，直到这行尾的，之间的部份都是注释，你完全可以把注释写在代码的头部，甚至可以在代码中间加上注释。

  • 单行注释格式：#注释内容
  • 多行注释格式：''' 注释内容 ''' （提示：三引号中的引号可以是单引号也可以是双引号）

运算符的优先级

运算符存在高低级别，在同一个表达式中，高优先级的先运算，对于同级别的运算符，按从左到右处理。

运算符优先级由低到高是：
Lambda
逻辑运算：or
逻辑运算：and
逻辑运算：not
成员测试：in,not in
同一性测试：is ,is not
比较：<,<=,>,>=,!=,==
按位或：|
按位异或:^
按位与：&
移位：<< ，>> （对二进制数字的移位）
加法与减法：+,-
乘法、除法与取余：，/，%
正负号：+x， -x
按位翻转：~x
指数：*

总结：

  • 运算符通常由左向右结合，具有相同优先级的运算符按照从左向右的顺序计算。例如：2+3+4被计算成（2+3）+4 。具有高优先级的，先计算高优先级的，再计算低优先级的，就如同先计算()括号里的，再计算()括号外的。
  • 而赋值运算符则是由右向左结合的，即a=b=c被处理为a=(b=c)
